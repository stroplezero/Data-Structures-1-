# 피보나치 수열 순환적 방법과 재귀적 방법 분석 및 비교
## 1. 개요
본 md는 피보나치 수열을 **재귀적 방법**과 **순환적 방법**으로 구현하고, 정수 N이 1부터 차례대로 증가함에 따른 수행 시간을 비교 분석한 결과이다.

## 2. 코드 구현
### 2-1. 재귀적 방법
시간 복잡도: O(2^n)
링크: [https://github.com/stroplezero/Data-Structures-1-/blob/main/project-8-1/main.c]

### 2-2. 순환적 방법
시간 복잡도: O(n)
링크: [https://github.com/stroplezero/Data-Structures-1-/blob/main/project-8-2/main.c]

## 3. 수행 시간 프로파일링 결과 
각 방법을 최대로 반복하기 위해 재귀적 방법은 N을 100으로 잡아 컴퓨터가 느려지는 최대치까지 수행해보았고, 순환적 방법은 N을 각각 100과 1000으로 잡아 수행 시간을 프로파일링해보았다.

## 4. 결과 정리
위의 두 방법에 따른 프로파일링 결과를 정리해보면 다음과 같다.  

| 방법 | 횟수 | 시간 | 증가랑 | 증가 배수 |
| ------ | ---- | ---- | ------ | ------|
| 재귀 | 50->52 | 263 -> 787 | +524 | 약 1.99배 |
| 순환 | 100->1000 | 4.949 -> 4.092 | -0.857 | 약 0.17배 |

## 5. 결과 해석 
위의 표를 보면 재귀적 방법으로는 N이 2만 증가해도 약 1.99배로 시간이 증가하는 것을 알 수 있고, 재귀적 방법은 O(2^n)의 시간 복잡도를 가지기 때문에 재귀적 방법으로는 52번째 수를 계산하는데에 컴퓨터의 연산으로도 13분이 걸림을 알 수 있다.  
순환적 방법은 N이 10배 증가해도 시간의 변화가 없는 것을 알 수 있다. 프로파일링에서 약 4초의 시간은 컴퓨터가 준비하는 기본 시간이 포함되어 있기 때문에 순환적 방법으로 실행할 때에는 N이 100에서 1000이어도 컴퓨터의 연산으로는 몇초에 불과하다는 것을 알 수 있다. 

## 5. 결론
재귀적 방법: N이 증가함에 따라 동일한 하위 문제를 중복해서 계산하므로 시간이 지수적으로 증가하며, O(2^n)의 시간 복잡도를 가져 N=52 이후로 시간이 눈에 띄도록 많이 소요됨. O(2^n)의 시간복잡도를 가진 문제는 컴퓨터의 연산으로도 소요 시간이 지수적으로 증가하여 컴퓨터로 O(2^n)의 시간복잡도를 가진 문제를 구할 수 없는 이유를 알 수 있었음.   
반복적 방법: N에 비례하여 선형적으로 시간이 증가하며, O(n)의 시간 복잡도를 가짐. N=50 이상의 큰 값에서도 매우 안정적인 성능을 보임. 컴퓨터의 연산으로는 N=100 과 N=1000 일 때의 연산 속도가 크게 차이나지 않음.
