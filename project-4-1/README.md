유클리드 호제법으로 GCD를 구하는 알고리즘의 시간 복잡도는 가장 핵심적인 코드 한 줄이 최악의 경우, 몇 번 실행되는지 알아보는 것으로 계산할 수 있다.
알고리즘에서 가장 핵심적인 코드 한줄은 main.c의 17번째 줄로 my_math.c의 코드가 최악의 경우 몇 번 실행되는지 알아보면 된다.
유클리드 호제법으로 GCD를 구하는 알고리즘의 최악의 경우는 i, j가 연속하는 피보나치 수일 때 이다.
연속하는 피보나치 수를 대입하여 핵심 코드가 몇 번 실행되는지 알아보면, 
피보나치 수열의 n번째 숫자가 F(n)이라 할 때, n을 5부터 순서대로 증가시키며 GCD(F(n), F(n-1))을 계산하면
n = 5일 때 3번, n = 6일 때 4번, n = 7일 때 5번 반복으로 n-2번 반복하는 것을 알 수 있다. 
따라서 f(n) = 4n-8로 Big-O를 계산하면 O(n)으로 나타난다.
